# Decisiones de arquitectura (ADRs)

Cada ADR documenta una decisión tomada, su contexto y sus consecuencias.
Se siguen estrictamente por orden cronológico y no se modifican
retroactivamente: si una decisión cambia, se añade una nueva ADR que
marque la anterior como **Superseded**.
